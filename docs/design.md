# Sunoパ！制作管理ツール 設計図

## 目的

Sunoパ！の放送準備から記事化、SNS投稿、画像制作、音源管理までをひとつの管理画面で扱えるようにする。

このツールは、Codexへ毎回バラバラに情報を渡すのではなく、放送回ごとに必要情報を集約し、記事作成に必要な「制作パック」を作ることを目的とする。

## 作りたいもの

```text
Sunoパ！制作管理ツール
├─ 放送回管理
├─ 複数フォーム作成
├─ 回答管理
├─ ゲスト情報管理
├─ リスナー応募曲管理
├─ パーソナリティ曲管理
├─ サムネ/画像制作管理
├─ 音源ファイル管理
├─ 記事作成パック生成
├─ SNS投稿文生成
└─ 将来: ポン出しアプリ連携
```

## 基本思想

- スプレッドシートやフォーム回答を「正本」として扱う。
- 記事に出す情報と、運営だけが見る情報を分ける。
- ゲスト回、通常回、リスナー応募回を同じ仕組みで扱えるようにする。
- Codexに渡す情報は、必要項目だけを抽出して整理する。
- 画像や音源は、放送回単位で整理する。
- 最初から全部自動化しない。まずは制作パックを作るところまでを確実にする。

## MVPで作る範囲


## 質問項目の設計

質問ごとに属性を持たせる。

```text
question_id
質問文
回答形式: 短文 / 長文 / URL / 選択式 / チェックボックス / ファイル
必須
公開してOKなプロフィール
記事で紹介してほしい内容
記事/SNSで触れないこと・表記ルール
制作側だけに共有するメモ
SNS投稿に使ってOK
漫画/画像案に使ってOK
表示順
```

この分類が重要。

```text
公開してOKなプロフィール: 記事やSNSの紹介文に使ってよい基本情報
記事で紹介してほしい内容: 本文の中心素材にするエピソードや楽曲の想い
記事/SNSで触れないこと・表記ルール: NG話題、表記、関係性、肩書きなどの厳守ルール
制作側だけに共有するメモ: 記事本文には載せない確認用メモ
SNS投稿に使ってOK / 漫画・画像案に使ってOK: 記事後の展開に使う素材
```

現MVPの共有フォームは、フォーム定義をURLに埋め込む試作方式。
回答者は入力後に回答JSONをコピーまたはダウンロードし、運営側が回答管理画面から読み込む。
Google Sheets/Driveへの自動回収は次フェーズで実装する。

```text

guest_public_profile.md
tracks_for_article.csv
article_constraints.md
thumbnail_assets.md
codex_request.md
```

含めるもの。

```text
stand.fm URL
ゲスト紹介
公開OK情報
紹介楽曲
埋め込みURL
記事に使う重要トピック
表記ルール
敬称ルール
NG/触れない情報
サムネURL
WordPress投稿先
```

含めないもの。

```text
個人情報
メールアドレス
内部だけのメモ
記事に載せないNG回答の本文

```

NG情報は本文をそのまま渡すのではなく、制約として渡す。

例:

```text
この記事では〇〇の話題に触れない。
このイベントの主催者は〇〇さん。
Silfiraは参加ではなくプロデュース。
```

## 判断メモ

- Googleフォームではなく自前フォームを優先する。理由は、質問ごとに「記事に使う」「内部確認のみ」「NG」などの属性を持たせたいから。
- スプレッドシートは正本として有効だが、長期的にはツール内DBを正本にする方が管理しやすい。
- 画像生成は後回し。まずは素材と制作ステータスを管理する。
- 音源のZIP一括DLは主導線にしない。放送回フォルダー整理を主導線にする。
- SE_Pon連携は後段。まずは取り込みデータの仕様だけ意識しておく。
